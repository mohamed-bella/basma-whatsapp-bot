Basma WhatsApp API Documentation

This document explains how to send messages to your clients from any external website, CRM, or script using HTTP Requests.

Authentication

All requests require an API key to ensure only your website/admin script can send messages.

Pass your secret key in the HTTP Headers:

x-api-key: basma_api_secret_2024

(You can change this secret inside your .env file)

Check Connection & QR Status

GET/api/qr

Returns the current connection status and a Base64 QR code image if disconnected.

Response (Success):
{
  "success": true,
  "status": "connected" // or "disconnected"
  "qr": null            // or "data:image/png;base64,iVBOR..."
}

1. Send Custom Text Message

POST/api/send-message

Sends a standard text message to any WhatsApp number. Great for custom alerts or admin notifications.

Field Type Description
phone String Req Client's full phone number with country code (e.g., "212704969534")
message String Req The message text. Supports bold (*text*), italic (_text_), and newlines (\n) 
// JavaScript Example
fetch("http://localhost:3000/api/send-message", {
  method: "POST",
  headers: { "Content-Type": "application/json", "x-api-key": "basma_api_secret_2024" },
  body: JSON.stringify({
    phone: "212704969534",
    message: "👋 Hello Admin! A new user just signed up."
  })
});

2. Send Order Confirmation

POST/api/send-order-message

Sends a beautifully formatted order receipt directly to the customer and saves the order in the bot's database.

Field Type Description
phone String Req Full phone number (e.g. "212704969534")
name String Req Customer Name
order_id String Req Unique Order ID (e.g. "1254")
product String Req Name of the Tour / Product
price String Req Price formatted (e.g. "200€")
date String Opt Tour date. E.g "12 April 2025" 
// PHP / WooCommerce Example
$data = array(
    'phone' => '212704969534',
    'name' => 'Ahmed',
    'order_id' => '1254',
    'product' => 'Sahara Desert Tour',
    'price' => '200€'
);

$ch = curl_init('http://localhost:3000/api/send-order-message');
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($data));
curl_setopt($ch, CURLOPT_HTTPHEADER, array(
    'Content-Type: application/json',
    'x-api-key: basma_api_secret_2024'
));
$response = curl_exec($ch);
curl_close($ch);

3. Update Order Status

PATCH/api/orders/:order_id/status

Updates an order status inside the bot database. Optionally sends a WhatsApp alert to the user notifying them of the change.

Field Type Description
status String Req Must be exactly: "Confirmed", "Pending", "Cancelled", or "Completed"
notify Boolean Opt If true, the bot immediately texts the user about this status change. 
// Node.js Example
fetch("http://localhost:3000/api/orders/1254/status", {
  method: "PATCH",
  headers: { "Content-Type": "application/json", "x-api-key": "basma_api_secret_2024" },
  body: JSON.stringify({
    status: "Cancelled",
    notify: true // Customer will instantly get a "Your order is cancelled" WhatsApp
  })
});

4. Advanced: Read Database

You can fetch the bot's stored data to display it on an external Admin Dashboard.

GET/api/orders

Returns all saved orders.

GET/api/users

Returns all users who have ever texted the bot, including their last active timestamp.
